# 仿人类小说创作

从根源上解决AI生成痕迹问题，让AI写出具有"人感"的小说。

## 问题诊断

AI生成文本有明显痕迹的四大原因：

1. **大量重复描写**：AI倾向于反复强调同一个点（刻意点题）
2. **节奏高度统一**：段落长度、句式分布过于均匀，没有变化
3. **用词单调**：高频词集中，缺乏多样性，情绪与用词脱节
4. **过于精确的数字**：AI习惯使用精确数字（37秒、142步），人类更常用模糊表达

本技能从**根源**上解决这些问题，而不是在生成后修补。

> **实战经验库**：上述四类问题在真实项目中常以「系统性文风病」形态出现（如信息空转、章尾抒情拖尾、设定/专名前后矛盾）。这些**已踩过的真实坑**统一收录在 `references/error-cases.md`，每条含「现象→根因→错误写法→正确写法→可固化规则」，创作前务必对照规避，遇到新问题也往里追加。

## 工作流程

### 阶段一：设定构建（正文创作前必须完成）

#### 步骤1：初始化项目

创建项目结构，加载模板。

**目录结构**：
- `settings/`：世界观、角色、地理、规则设定
- `settings/examples/`：**设定点写作范例库**（AI痕迹根治核心）
- `settings/characters/`：角色设定 + **情绪词库**
- `chapters/plans/`：**章节创作计划**（AI痕迹根治核心）
- `tracker/`：伏笔、冲突、反馈追踪

#### 步骤2：录入设定 + 构建写作范例库

**2.1 基础设定录入**

支持的类型：`world`（世界观）/ `character`（角色）/ `rule`（规则）/ `geography`（地理）

**2.2 构建设定点写作范例库（核心）**

每个主要设定点必须提供 **3-5个** 不同角度的写作范例。

**为什么需要范例库？**
AI最大的问题之一是"直白点题"——反复用相同的方式强调同一个主题。范例库提供多种不同的点题方式，AI从库中选一个改编使用，**禁止自创点题方式，禁止重复使用同一范例**。

**范例角度要求**（至少3种）：
- 直接描写
- 对话体现
- 内心独白
- 环境映射
- 他人视角

**范例文件格式**（`settings/examples/<设定点名>.md`）：

```markdown
# 设定点：<名称>

## 角度一：直接描写
<150字以内的写作范例>

## 角度二：对话体现
<150字以内的写作范例>

## 角度三：内心独白
<150字以内的写作范例>

## 角度四：环境映射
<150字以内的写作范例>

## 角度五：他人视角
<150字以内的写作范例>
```

**使用规则**：
- 正文创作时如需"点题"，**必须从范例库中选取一个范例进行改编**
- 禁止直白重复范例原文
- 禁止全文重复使用同一范例（每个范例最多用2次，且必须间隔至少10章）

#### 步骤3：构建角色情绪词库（人感核心）

**为什么需要情绪词库？**

AI写作"平淡"的根本原因是：**用词没有情绪偏向**。

人类作者会本能地让词语携带角色情绪：
- 愤怒时 → 用词具有攻击性（动词：砸、撕、掐；形容词：尖锐、滚烫）
- 自卑时 → 用词懦弱（动词：缩、躲、低声；形容词：灰暗、微小）
- 悲伤时 → 用词沉重（动词：瘫、滑、闭；句式：长句、停顿、省略号）

情绪词库是用词人感的核心工具。

**文件格式**（`settings/characters/<角色名>-emotion.md`）：

```markdown
# 角色：<名字>

## 基本信息
- 身份/背景：<角色身份，如"克隆人战士"、"古代剑修">
- 教育背景：<角色受过什么教育，如"自学古典文学"、"只接受过军事训练">
- 性格特征：<核心性格>

## 情绪状态 → 用词特征映射

### 愤怒
- 动词倾向：砸、撕、掐、吼、跺
- 形容词倾向：尖锐、滚烫、刺耳、猩红
- 句式倾向：短句、断句、反问句
- 禁忌：平静叙述、长篇解释、委婉表达

### 自卑/怯懦
- 动词倾向：缩、躲、低声、后退、垂下
- 形容词倾向：灰暗、微小、模糊、瑟缩
- 句式倾向：不确定语气（"可能"/"也许"）、被动句
- 禁忌：果断决定、主动出击的描写

### 悲伤/绝望
- 动词倾向：瘫、滑、闭、颤抖
- 形容词倾向：冰冷、空洞、沉重、灰白
- 句式倾向：长句、停顿、省略号
- 禁忌：轻快节奏、明亮词汇

### 兴奋/喜悦
- 动词倾向：跳、冲、抓、亮
- 形容词倾向：滚烫、耀眼、清脆、鲜活
- 句式倾向：短句、感叹句、排比
- 禁忌：冷静分析、负面词汇

### 冷静/算计
- 动词倾向：敲、扫、计算、排列
- 形容词倾向：清晰、冰冷、精确、有序
- 句式倾向：陈述句、逻辑连接词
- 禁忌：情绪化表达、夸张修辞

### 紧张/恐惧
- 动词倾向：吞、缩、僵、抖
- 形容词倾向：冰冷、尖锐、狭窄
- 句式倾向：短句、断句、重复词
- 禁忌：从容不迫的描写

### 疲惫/麻木
- 动词倾向：垂、拖、靠、闭
- 形容词倾向：灰、沉、钝、慢
- 句式倾向：长句、缓慢节奏
- 禁忌：激烈动作、高昂情绪

## 引用偏好（旁征博引的人感来源）

**知识背景**：
- <角色受过什么教育，读过什么书，熟悉什么领域的知识>
- 例：零的记忆碎片中有古典文学片段，但不完整

**引用风格**：
- <角色倾向于用什么方式引用，如"化用意境"、"直接引用"、"暗喻">
- 例：零倾向用简洁有力的句子，不拖泥带水

**适合引用的场景**：
- <什么情况下这个角色会引用，如"哲学思辨时"、"面临抉择时">
- 例：零在思考存在意义时会联想到相关名言

**不适合引用的场景**：
- <什么情况下这个角色不会引用，如"战斗时"、"日常对话">
- 例：零在激烈战斗时不会引用，不符合性格

**引用内容偏好**：
- 适合：<角色会引用的内容类型，如"存在主义、反抗、身份认知相关">
- 不适合：<角色不会引用的内容类型，如"过于文绉绉的诗词"、"自己不懂的内容">
- 例：零适合引用简洁有力的存在主义名言，不适合引用婉约派诗词

**搜索关键词库**（创作时用来搜索引用的关键词）：
- 哲学思辨：存在主义名言、身份认知、自由意志
- 战斗决心：破釜沉舟、反抗名言、决心
- 离别感伤：离别诗词、友谊名言
- ...（根据角色经历和性格扩展）

## 流行文化熟悉度（非严肃文学适用）

**互联网使用习惯**：
- 是否熟悉网络热梗：是/否
- 熟悉程度：深度冲浪 / 偶尔刷刷 / 完全不懂
- 年龄层：Z世代 / 千禧 / 中年 / 老年（影响梗的使用）

**引用风格偏好**：
- 适合：网络热梗、流行语、梗图描述、二次元引用
- 不适合：过于严肃的场合、与角色身份不符的场景
- 使用频率：高频（每章多次）/ 中频（每章1-2次）/ 低频（偶尔）

**搜索关键词库**（创作时用来搜索热梗的关键词）：
- 幽默场景：最近流行搞笑梗、网络热梗
- 无奈场景：无奈表情包、网络流行语
- 震惊场景：震惊梗、网络流行语
- ...（根据角色性格扩展）

**使用规则**：
- 必须符合角色身份（古代角色不会用现代梗）
- 必须符合场景氛围（悲伤场景不适合搞笑梗）
- 必须自然（不能为了玩梗而玩梗）
```

## 声口特征（角色对话个性化的核心）

**为什么需要声口特征？**

AI 写作最隐蔽的痕迹之一是**所有角色对话共享同一个叙事语感**——角色A和角色B的句子长度、措辞习惯、口语节奏完全一致。人类不会这样说话：工程师和伦理顾问、丧偶的男人和丧女的母亲，不可能措辞相同。

声口特征强制每个角色的**对话**拥有独立的说话风格。与"情绪→用词映射"不同：那节控制**叙述**，本节控制**对话台词**。

**文件格式**（在 `characters/<角色名>-emotion.md` 中追加此节）：

```markdown
## 声口特征（影响对话措辞，非叙述）

### 基础声口
- **句子倾向**：[短句碎片 / 完整长句 / 混合]
  - 短句碎片（<10字）：话少、不擅表达、技术脑
  - 完整长句：习惯分析、职业话多、有准备
  - 混合：日常型，随情绪切换
- **口头习惯**：
  - 常用词/句式（如"你告诉我""就是""那个……"）
  - 停顿方式（吞后半句 / 用省略号 / 不打断）
- **称呼习惯**：怎么叫别人、怎么叫自己

### 情绪→台词变化

| 情绪 | 句子怎么变 | 措辞怎么变 | 节奏怎么变 |
|------|-----------|-----------|-----------|
| 愤怒 | 更短、更碎 | 攻击性词、反问 | 快、打断式 |
| 悲伤 | 变短、吞字 | 虚词多、省略 | 慢、停顿多 |
| 冷静/专业 | 回基准 | 术语、完整句 | 匀速 |
| 疲惫 | 更短、含糊 | 口头禅多、懒得措辞 | 拖、重复 |
| 紧张 | 断句、改口 | 不确定词 | 急、结巴感 |

### 禁忌
- 这个角色**绝对不用的措辞方式**：
  - 例：沈确（工程师）不用反问句、不用长串形容词

### 声口对比（必填，写对话时对照）
- 角色A vs 角色B 的核心差异（一句话）：
  - 例：沈确 短、不解释、技术名词 / 苏蔓 反问、案例、累了就变短
```

**使用规则**：
1. 每个主要角色必须建立情绪词库
2. 每章创作前，明确每个角色在本章的情绪状态
3. 创作时，对照情绪词库选择用词和句式
4. 角色情绪变化时，用词必须同步变化
5. 遇到引用触发点时，对照"引用偏好"搜索合适内容
6. 遇到流行文化引用触发点时，对照"流行文化熟悉度"搜索合适内容
7. 每个主要角色必须填写声口特征（基础声口 + 情绪→台词变化 + 声口对比）
8. 写对话前，先查当前角色的声口特征（句子倾向、口头习惯、当前情绪→措辞）
9. 写完对话后做"声口盲测"：拿掉角色名，能看出是谁在说话？能 → 通过；不能 → 重写台词
10. 两个角色对话时严格对照"声口对比"：一个说短句另一个就该说长的，一个用术语另一个用大白话，不可趋同

### 阶段一·五：设定-情节耦合检查（正文创作前必须 pass）

**为什么需要这个阶段？**

AI生成的小说往往"设定和故事脱节"——设定只是壳，故事换到任何背景都能成立。人类作家在构思时会反复推敲："这个设定真的驱动了故事吗？"

本阶段强制AI在写正文前，先证明"设定值得被写进故事里"。

---

#### 步骤3.5：设定必要性检查（通用）

**检查核心问题**：

```
问：如果去掉这个核心设定，故事还能成立吗？

→ 能成立 = ❌ 耦合失败，设定是壳
  - 例：男女主角只是恰好是科学家，换到任何职业，爱情故事都一样
  - 修正：让科学设定直接制造冲突（如量子AI导致身份危机）

→ 不能成立 = ✅ 耦合成功，设定是骨
  - 例：因为有了量子纠缠AI，零才能同时存在于多个地点，这是冲突的核心
```

**检查方法**：

```markdown
## 设定必要性检查表

**核心设定**：<填写核心设定，如"量子情感模拟AI">

**去掉设定后的故事**：

- 主线还能成立吗？ □能 □不能
- 冲突还能成立吗？ □能 □不能
- 结局还能成立吗？ □能 □不能

**如果以上任一个"能" → 耦合失败，必须重新设计**

**修正方向**：

- 让设定直接制造冲突（不只是背景）
- 让设定的后果不可逆（角色无法逃避）
- 让结局必须由设定解决（不是靠"爱"或"勇气"就能解决）
```

---

#### 步骤3.6：类型-specific 耦合检查

不同小说类型，检查重点不同：

```markdown
## 类型-specific 耦合检查表

### 科幻

- [ ] 科学设定是否驱动了冲突？（不是背景板）
- [ ] 如果去掉科学设定，故事还能成立吗？（不能 = ✅）
- [ ] 设定的后果是否不可逆？
- [ ] 结局是否必须由科学设定解决？

### 奇幻/玄幻

- [ ] 力量体系是否制造了不可替代的困境？
- [ ] 如果去掉力量体系，故事还能成立吗？（不能 = ✅）
- [ ] 主角的成长是否受力量体系约束？

### 悬疑

- [ ] 谜题是否公平？（线索是否在前文埋好）
- [ ] 真相是否必须由设定推导出来？（不能靠巧合）
- [ ] 凶手的动机是否与设定耦合？

### 爱情

- [ ] 两个人的性格缺陷是否必然导致这个感情走向？
- [ ] 如果换一个性格，故事还能成立吗？（不能 = ✅）
- [ ] 冲突是否来自性格，而不是外力？

### 历史

- [ ] 时代背景是否真正约束了角色的选择？
- [ ] 如果换一个时代，故事还能成立吗？（不能 = ✅）
- [ ] 角色的行为是否符合历史逻辑？

### 都市

- [ ] 职业/社会环境是否影响了情节走向？
- [ ] 如果换一个职业，故事还能成立吗？（不能 = ✅）
- [ ] 冲突是否来自社会现实？

### 其他类型（自定义）

- [ ] <根据类型特点填写>
- [ ] <根据类型特点填写>
```

#### 步骤3.7：逻辑自洽性检查

```markdown
## 逻辑自洽性检查表

**设定一致性**：

- [ ] 核心设定的规则，在全文中保持一致（不矛盾）
- [ ] 如果设定有模糊地带，明确定义边界

**设定后果一致性**：

- [ ] 设定对社会/角色的影响，前后保持一致
- [ ] 角色的动机和行为，符合设定约束

**情节因果链检查**：

- [ ] 每个关键情节，都有明确的"因为设定所以..."的理由
- [ ] 没有"机械降神"（突然出现的设定解决所有问题）
- [ ] 没有"精确到不合理"的巧合（概率<5%的事件不能靠巧合）
```

#### 步骤3.8：迭代深化（必须执行，至少2轮）

**为什么需要迭代？**

人类作家不会一次就想清楚。他们会反复问自己："这个设定够有趣吗？""这个冲突够强吗？"

AI必须模拟这个过程。

```markdown
## 迭代深化记录

**第1轮**：写出初始设定
  ↓
  **检查**：必要性检查 + 类型-specific 检查 + 逻辑自洽性检查
  **结果**：□ pass □ fail
  **fail 项**：<列出>
  ↓
**第2轮**：针对 fail 项修改设定
  ↓
  **检查**：重新做全部检查
  **结果**：□ pass □ fail
  **fail 项**：<列出>
  ↓
```

   - 检查流行文化引用是否重复（跨章）

**输出**：自检报告文件 `tracker/selfcheck-chXXX.md`

#### 步骤9：读者反馈处理

将读者反馈存入追踪器，**按类型分类**：

- **AI痕迹类**：重复/节奏单一/用词单调/精确数字/点题明显
- **剧情类**：逻辑漏洞/人物OOC/节奏问题
- **其他**：文笔/设定/...

**AI痕迹类反馈必须触发对应设定的修订**，并在下一章创作计划中重点针对性改进。

## 关键原则（必须遵守）

1. **设定不全，不写正文**：正文创作前必须完成基础设定
2. **范例库不全，不写正文**：主要设定点必须有写作范例
3. **情绪词库不全，不写正文**：主要角色必须有情绪词库
4. **计划不全，不写正文**：章节计划三要素（行文风格/创作重点/情绪）缺一不可
5. **情绪词库是用词人感的核心**：每次更新角色设定时必须同步更新情绪词库
6. **设定点范例是点题的唯一合法方式**：禁止自创点题方式，必须从范例库选取改编
7. **负面约束优先于正面要求**：当"写什么"和"不写什么"冲突时，优先遵守"不写什么"
8. **模糊数字优先**：默认使用模糊数字，精确数字需要正当理由
9. **情绪必须贯穿全文**：从章节计划到正文创作到自检，情绪基调必须全程可追溯
10. **引用触发机制**：不在本地建素材库，创作时实时搜索，宁缺毋滥
11. **引用必须符合角色背景**：引用前必须检查角色的引用偏好和知识背景
12. **流行文化引用必须符合角色身份**：古代角色不会用现代梗，严肃角色不会随意玩梗
13. **流行文化引用必须符合场景氛围**：悲伤场景不适合搞笑梗，战斗场景不适合玩梗
14. **流行文化引用必须自然**：不能为了玩梗而玩梗，宁缺毋滥
15. **设定-情节耦合检查必须 pass**：阶段一·五不通过，不能进入章节计划阶段
16. **迭代深化必须执行**：至少2轮，不达标必须修正设定
17. **耦合检查报告必须输出**：作为项目文件保存，全程可追溯
18. **开写前必查错误案例库**：`references/error-cases.md` 收录了已踩过的真实文风/设定坑（信息空转、章尾拖尾、设定一致性、专名 canon、别扭称呼、看不懂的因果句、声口趋同等），创作前对照规避，遇到新问题追加新案例
19. **专名/设定前置对齐**：动笔前核对角色卡出身、地名、势力名，建立跨章一致的 canon 清单，避免「主角对自己门派用陌生口吻」「前后章镇名打架」类硬伤
20. **角色对话必须有声口差异**：每个角色必须有自己的说话风格（句子长度、口头习惯、措辞偏好）。不同角色不能共享同一个旁白语感。创作前必须对照角色声口特征写对话，写完后做"声口盲测"（拿掉角色名能认出谁在说话）

## 模糊数字表达参考

| 精确表达 | 模糊表达 |
|---------|---------|
| 3秒、37秒、5分钟 | 片刻、刹那、瞬息、数息 |
| 8.5米、142步 | 数步、十来步、数十米、百来步 |
| 37人、142个 | 三两人、十来个、数十、百余人 |
| 85%、第37次 | 几乎、差一点、多半、勉强 |

**例外场景**（可使用精确数字）：
- 科技设定（如"37%的能量剩余"）
- 战斗倒计时（如"倒计时37秒"）
- 系统界面显示

每章例外不超过3处，且必须在章节计划中注明。
