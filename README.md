[![GitHub Repo stars](https://img.shields.io/github/stars/LittleLollipop/human-like-novel-workbuddy-skill?style=social)](https://github.com/LittleLollipop/human-like-novel-workbuddy-skill)

[![Ko-fi](https://img.shields.io/badge/Ko--fi-支持作者-ff5f5f?style=flat&logo=ko-fi)](https://ko-fi.com/sai1990)

从根源上解决AI生成痕迹问题，让AI写出具有"人感"的小说。

## 问题痛点

AI生成的小说往往有明显的"AI味"，主要体现在：

1. **大量重复描写**（刻意点题）
2. **行文节奏高度统一**，没有变化
3. **用词单调**，缺乏情绪倾向
4. **过于精确的数字**
5. **缺乏旁征博引**（名言/典故/诗词）
6. **缺乏流行文化引用**（网络热梗/搞笑梗）

## 解决方案

本技能通过以下机制，从根源上解决上述问题：

### 1. 情绪词库（人感核心）

每个角色每种情绪状态都有对应的**动词、形容词、句式倾向**。

**示例**：
- 愤怒时 → 动词：砸、撕、掐、吼；形容词：尖锐、滚烫；句式：短句、断句
- 悲伤时 → 动词：瘫、滑、闭；形容词：冰冷、空洞；句式：长句、省略号

**效果**：AI写作时会本能地让词语携带角色情绪，就像人类作者一样。

---

### 2. 设定点范例库（解决重复点题）

每个主要设定点必须有 **3-5个多角度写作范例**。

**示例**：
- 设定点："零的执念"
- 角度一：直接心理描写
- 角度二：对话体现
- 角度三：内心独白
- 角度四：环境映射
- 角度五：他人视角

**效果**：正文创作时如需"点题"，必须从范例库选一个**改编**。禁止自创点题方式，禁止全文重复使用同一范例。

---

### 3. 章节三要素计划（解决节奏单一）

每章写正文前，必须生成详细的**创作计划**，包含三要素：

#### 要素①：行文风格
- 节奏分布：[快-慢-快] / [慢-快-慢] / [匀速紧张] / [舒缓铺陈]
- 句式混合要求：单句段落至少N处，长段落不超过N处
- 章节开头方式（必须与前章不同）
- 章节结尾方式（禁止总结性点题）

#### 要素②：创作重点
- 本章核心偏向：感情戏 / 剧情推进 / 世界背景介绍 / 环境描写 / 动作场面 / 内心戏 / 对话戏
- 内容比重建议

#### 要素③：情绪基调（人感核心）
- 本章整体情绪
- 角色情绪状态（影响用词选择）
- 用词约束

---

### 4. 负面约束系统（解决AI痕迹）

每章创作时，必须逐条检查以下负面约束：

- [ ] 禁止直白点题
- [ ] 禁止精确数字（例外：科技设定/战斗倒计时，每章不超过3处）
- [ ] 禁止重复前章用词
- [ ] 禁止连续3段相同句式
- [ ] 禁止节奏均匀分布

---

### 5. 引用触发机制（解决旁征博引问题）

**不在本地建素材库**，创作时实时搜索。

#### 引用触发点（在章节计划中定义）

```markdown
**触发点1**：零在废墟中思考存在意义
- 触发条件：哲学思辨场景
- 搜索方向：存在主义名言、身份认知
- 引用风格：内心独白，化用，不能直白
- 角色适配：零（记忆碎片中有古典文学片段）
```

#### 创作时的引用触发检查

每段写完，检查是否符合引用触发点：
1. 如果符合，调用 WebSearch 搜索相关名言/典故/诗词
2. 将引用**化用**进文本（用意境，不直接抄）
3. 检查引用是否自然，如果生硬则放弃

**原则**：宁缺毋滥，找不到合适的就不引用。

---

### 6. 流行文化引用触发机制（非严肃文学适用）

在轻松/幽默场景引用**网络热梗、流行语**，增加时代感和亲和力。

#### 角色设定中的流行文化熟悉度

```markdown
## 流行文化熟悉度

**互联网使用习惯**：
- 是否熟悉网络热梗：是
- 熟悉程度：深度冲浪
- 年龄层：Z世代

**使用频率**：中频（每章1-2次）
```

#### 流行文化引用触发点

```markdown
**触发点1**：角色遇到无奈的事
- 触发条件：幽默场景
- 搜索方向：最近流行无奈梗、网络流行语
- 引用风格：角色内心吐槽、对话中自然带出
```

**原则**：
- 必须符合角色身份（古代角色不会用现代梗）
- 必须符合场景氛围（悲伤场景不适合搞笑梗）
- 必须自然（不能为了玩梗而玩梗）

## 使用方法

### 完整工作流程

```
1. 开书初始化
   → 创建项目结构

2. 建设定（阶段一）
   → 录入世界观、角色设定
   → 为每个主要设定点创建写作范例（3-5个角度）
   → 为每个主要角色创建情绪词库
   → 填写角色引用偏好和流行文化熟悉度

3. 写章节计划（阶段二，正文创作前必须）
   → 为每章生成三要素计划
   → 定义引用触发点
   → 定义流行文化引用触发点（非严肃文学）
   → 填写强制用词清单

4. 写正文（阶段三）
   → 按三要素创作
   → 每段写完执行词库查阅检查
   → 每段写完执行引用触发检查
   → 每段写完执行流行文化引用触发检查
   → 每段写完执行负面约束检查
   → 输出时附加创作说明

5. 章节自检（阶段四）
   → 自动检测AI痕迹
   → 生成自检报告
```

---

### 命令行工具（可选）

本技能提供 `novel.py` 命令行工具，帮助管理创作流程。

```bash
# 初始化项目
python ~/.workbuddy/skills/human-like-novel/scripts/novel.py init <书名>

# 录入设定
python ~/.workbuddy/skills/human-like-novel/scripts/novel.py setting <类型> <内容>

# 创建设定点范例
python ~/.workbuddy/skills/human-like-novel/scripts/novel.py example <设定点名> <范例内容>

# 创建章节计划
python ~/.workbuddy/skills/human-like-novel/scripts/novel.py plan <章号>

# 章节自检
python ~/.workbuddy/skills/human-like-novel/scripts/novel.py selfcheck <章号>
```

## 核心原则（必须遵守）

1. **设定不全，不写正文**
2. **范例库不全，不写正文**
3. **情绪词库不全，不写正文**
4. **计划不全，不写正文**
5. **情绪词库是用词人感的核心**
6. **设定点范例是点题的唯一合法方式**
7. **负面约束优先于正面要求**
8. **模糊数字优先**
9. **情绪必须贯穿全文**
10. **引用触发机制**：不在本地建素材库，创作时实时搜索
11. **引用必须符合角色背景**
12. **流行文化引用必须符合角色身份**
13. **流行文化引用必须符合场景氛围**
14. **流行文化引用必须自然**

## 效果对比

| 问题 | 传统AI写作 | 使用本技能 |
|------|------------|------------|
| 重复描写 | ❌ 频繁出现 | ✅ 从范例库选角度，避免重复 |
| 节奏单一 | ❌ 全文匀速 | ✅ 章节计划约束节奏分布 |
| 用词单调 | ❌ 缺乏情绪倾向 | ✅ 情绪词库约束用词 |
| 精确数字 | ❌ 频繁出现 | ✅ 负面约束强制模糊化 |
| 缺乏引用 | ❌ 很少引用 | ✅ 引用触发机制主动搜索 |
| 缺乏流行文化 | ❌ 没有 | ✅ 流行文化引用触发机制 |
